System Parameters

 

DB Distributor > Item Management > Setup > Other > Setups and Parameters > System Parameters (FMS S1)

 

Setting the Item Management System Parameters is the first step in initializing your Item Management System for processing. These parameters allow you to customize the processing of the system to meet your information goals.

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Method for Calculating Average Monthly Usage?

Your options here are "Average" (straight average) or "Weighted Average". Weighted average is recommended. This should not be changed after it is set. Using the straight average will simply average the previous "X" number of months (with X being the number of months you choose in the following parameter). The weighted average will use the previous months average to count for X - 1 months, and have the current usage count for 1 month. Example: Using a 6 month weighted average with 3,500 as the previous months weighted average, and 4,200 as the current usage. ((3,500x(6-1))+4,200)/6=3,617. Essentially, in this example, the previous months weighted average accounts for 5 of the 6 months when calculating the new weighted average.
Period of time in months for calculation? This is the number of months that will be used for calculating the average monthly usage.
Number of months to dilute 1st shipments? This field represents the number of months the first shipment will be divided by to determine a starting usage calculation.
Number of periods to save distribution detail? This can be set between 1 and 99. The recommendation is 3 periods. This can be set higher if your DemandBridge server has available data storage capacity.
Number of years to save usage summary data? This can be set between 1 and 9. The recommendation is 3 years.
IMS Options You can enter an "F" for allowing fractional average usages. You can also enter an "S" to designate that stock items should not be updated as IM items.
Method of updating issues based usage?

This can be set to "Usage on Shipment" (recommended) or "Usage on Next Shipment". Click here for a detailed explanation.
Update issues based usage in current period? This can be set to Yes or No. Click here for a detailed explanation.
Update out-of-date usage all to current period? This can be set to Yes or No. The recommendation is no. Click here for a detailed explanation.
Update special charges to IMS pricing in FMZ? This can be set to Yes or No. If this is set to Yes, the IMS pricing fields in FMZ (Item Detail Activity file) will be updated for all Special charges during the invoice update.
Are reorder notices automatically generated? This can be set to Yes or No. The recommendation is No. Setting this to Yes will cause DB Distributor to automatically generate reorder notices as a result of completing the IM Cycle Usage Update. You should set this to No if you will be using the On-Demand Reorder Notice Update or manually specifying reorder notices.
Type of usage graph on reorder notices? The usage graph can be set to Current Year or Prior Months. Current Year will give the customer a graph based on their current year. Prior Months will give the customer a graph based on their past 12 months of usage history.
How long to keep inactive count stops (months)? This can be set between 1 and 9. The recommendation is 3 months. For counted locations, once a count is entered for an item, the system will continue to print an entry for the item on the location's count sheet. If the location stops using the item, no future counts will ever be entered. This option specifies the number of months that will pass, without a count being entered, before the system will remove the item from the locations count sheet.
Ask Update IFN print file question on "Inactive/Obsolete Item Analysis" screen? This can be set to Yes or No. If answered with a Yes, the Inactive / Obsolete Item Analysis Report will have a new Question on the bottom of the selection screen. “Update Inactive Item Notice Print File?” The Inactive Item Notice is a preprinted form that is used to notify customers of forms that are inactive.
