Accounts Receivable Parameters - Screen 4

 

DB Distributor > Accounts Receivable > Setup > System > System Setup > Parameters (ARS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Maximum number of salespeople per invoice

Normally you would only have one salesperson per invoice. However, you can have up to five salespeople sharing a commission. If you split commissions, during invoicing and customer setup, you will be required to enter the percentage of the total commission amount that each salesperson is to receive when splitting invoice commissions.
Enter code(s) of statistical data to be retained DB Distributor has the following statistical options for salespeople: Sales (billed), Profit (billed), Commissions (billed), Number of invoices, Quota, Booked Sales, Booked Profit, Adjusted Accumulated GP, and GP without adjustment. Hold the "ctrl" button on the keyboard and click on the options you would like to keep data for, and then press the "lock" button.
Enter the corresponding number of prior years statistics to keep This field will hold your preference for the number of prior years statistics to retain. Just enter a number from zero to nine, for each of the statistics you chose to keep in the previous question. The current year will always be kept for the selected codes. Once Statistical data has been purged, you will not be able to retrieve it, except from backup. It is better to enter three or four here and adjust it down if you find that you are not using the older statistics.
Report commissions when an invoice is paid? If you turn this "on", commission will write to the commissions "payable" file at the time an invoice is paid. This means that commissions will write to a "pending" file at the time an invoice is created. If this is turned "off", commissions will write directly to the "payable" file when an invoice is created, and there will not be any "pending" commission. This is usually checked "on".
Pay commission only when an invoice is paid in full? If the previous parameter is turned "on", you have the ability to tell the system to only pay commission when an invoice is fully paid. If this is not turned "on", commissions will be paid on a pro-rated bases as an invoice is paid off.
Do you have declining commission based on invoice ageing If you wish to pay declining commission based upon invoice ageing, you will want to turn this "on". You can setup the declining commission table in the following parameter. The default would be to decrease commission by paying only X percent of the available commission. You can also specify if you would rather reduce commission by decreasing the net sale, or the gross profit.
Declining commission table

On the salesperson record, you can assign them a "type" by assigning the numbers 0 - 4. In this table you can set what the percentage to be paid is based upon the selection you made in the previous parameter.
