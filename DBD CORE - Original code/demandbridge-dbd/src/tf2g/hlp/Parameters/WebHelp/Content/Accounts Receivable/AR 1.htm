Accounts Receivable Parameters - Screen 1

 

DB Distributor > Accounts Receivable > Setup > System > System Setup > Parameters (ARS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Current Fiscal Year

Enter in the current fiscal year. This will automatically update to the new year when you run the year-end process.
Current Accounting Period Enter in the current accounting period. This will automatically update to the new period when you run the period-end process.
Divisions Required? If your company has multiple locations or branches and would like to have different P&L statements on the financial reporting, set this to "yes".
What size customer code should be used? The Customer code can be up to eight alpha-numeric characters in length. It is recommended that a minimum of four characters be used. Enter your requirements for what size customer code should be used. If you specified divisions, the customer code will be prefixed by an additional two-digits. For instance, if you enter “5” and you check the “Divisions Required” box, your customer code would look like this, XX-XXXXX.
Alternate bill-to customer codes? Setting this to "yes" will give you the ability to tell the system that invoices for one customer should be paid by another customer.
Are there multiple bank codes? Check this box if you use more than one bank account for processing cash receipts.
What is the standard bank code?

When processing cash receipts, you will be required to enter a bank code. The system will default to the bank code you enter here. Normally, enter "A". Later, using the bank code setup program you will enter the specifics for this code.
Will the GL control accounts be interfaced by category? If your GL Accounts will be controlled by the Customer Category, answer "yes".
What is the maximum size of the customer category code? The customer category code may be from one to nine digits in length. If you are using the Contact Management System, you will need to set this value to three. Since it is an alphanumeric field, three should be adequate for all but extreme circumstances.
Enter code(s) of statistical data to be retained DB Distributor has the following statistical options for customers: Sales, Profit, Number of Invoices, Booked Orders, and Booked Profit from orders. Hold the "ctrl" button on the keyboard and click on the options you would like to keep data for, and then press the "lock" button.
Enter the corresponding number of prior years statistics to keep This field will hold your preference for the number of prior years statistics to retain. Just enter a number from zero to nine, for each of the statistics you chose to keep in the previous question. The current year will always be kept for the selected codes. Once Statistical data has been purged, you will not be able to retrieve it, except from backup. It is better to enter three or four here and adjust it down if you find that you are not using the older statistics.
Number of periods to retain invoice history Enter in the number of periods of invoice history to keep. This should be set to at least 24.
Print the GL distribution on the summary sales journal? If you would like for the GL postings to be displayed on the summary sales journal, you will set this to "yes". When updating your sales journal, you have the option of printing both the detail and summary sales journal. The detail sales journal always prints the GL postings. If you choose not to print the detail sales journal, then you will want to have this set to "yes" to make sure you still have access to that information.
