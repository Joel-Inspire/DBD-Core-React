Accounts Receivable Parameters - Screen 6

 

DB Distributor > Accounts Receivable > Setup > System > System Setup > Parameters (ARS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Enable credit requestor feature?

DB Distributor also provides a system for tracking credit references that you provide to potential vendors for your customers. Check this box to initiate this feature.
Comm/Payable to be affected by discounts taken in C/R? If you offer prompt pay discounts to your customers, and you want to reduce commissions when these discounts are taken, check this box. The commission will be reduced by the same percentage that the discount was to the gross profit. Note: This feature is only available for commissions paid upon collection.
Shipments to 'D' locations use sales tax code in whse loc rec? Certain taxing jurisdictions require that goods shipped to a warehouse be taxed based upon the tax code and rate of the jurisdiction where the warehouse is located. If this box is checked and there is no sales tax code in a particular warehouse, the tax code from the customer will be used. This provides you with the ability to have multiple warehouses in multiple tax jurisdictions and to only use the sales tax code from the warehouse when it is appropriate. Check with your tax advisors to determine if this parameter relates to you.
Interstate orders use tax codes from warehouse ship-to? If you check the box associated with the parameter question above. You can also differentiate from orders shipped within your own state. You can designate if these orders should also be taxed based upon the warehouse ship-to code or if these orders are exempt from this rule and should be taxed based on the code in the customer file.
Enable Credit Card Feature? The ability to accept credit cards is provided through an optional module that works in conjunction with eCommerce. If you have licensed this module, you will have access to this parameter and the other options associated with credit card processing.
Do you have a sales tax download service? DB Distributor offers an optional module that provides for interfaces to certain sales tax update services. If you have licensed this module, you will be able to have access to this parameter and the other associated options.
Enter the length for each segment of the sales tax number

The standard DB Distributor sales tax code displays in the following format: ##-####-####. The length of each segment of the above format would be 2-4-4. Certain update services will use their own format .
Sales tax numbers will be displayed in this format This field simply displays the format based upon the numbers you entered in the previous parameter.
Use lot std cost to create GP adj during invoicing? Salesperson gross profit can be calculated in a number of different ways. It can be the true accounting gross profit or it can be a reduced amount based upon one or more house charges. It can also be based upon a standard Salesperson Cost. The salesperson cost can be set at the item level or even down to the lot record. Check this box if you want salesperson gross profit and the associated commission to be based upon the salesperson cost stored at the lot level.
Std inv msg for credit card invoices You have the option to print confirming invoices for sales paid by credit cards. Enter the standard message that you would want to appear on these confirming invoices. “Paid in Full by Credit Card”, for example.
Enable ability to tax customer owned goods at time of release? If your customer is considered a reseller of product you provide, and if they are paying for product at the time the goods are shipped to your warehouse, they may be required to account for sales tax based upon the shipping destination at the time of release. By checking this box, sales tax will not be charged at the time the goods are received into your warehouse and your customer is invoiced, but at the time the goods are released from inventory. Since the products have already been paid for, the invoices and/or summary bills will only include the sales tax, freight or other shipping and handling charge.
Update commission stats during Invoice/Chargeback Entry? DB Distributor provides for variable commission rates based upon a table of gross profit. This can be done at the line item level, the invoice level or even based upon year-to-date statistics. If the later method is used, you will probably want commission charge backs to accrue to these year-to-date statistics files as well.
Update invoice entry batch files to allow ODBC access? In addition to the information that writes to the Invoice History file, all of the additional information available during the invoice entry process will update to the "AZ" files to be accessed by ODBC. These come in handy if you wish to make custom reports based off of billings.
Enable cash posting invoice range logic? This will allow you to clear a range of invoices during cash receipts.
