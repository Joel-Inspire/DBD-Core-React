Updating Unbilled Costs

 

This explanation covers the first 2 parameters on tab 8 of the Order Processing parameters. You may use the first parameter without the second, but you may not use the second parameter without the first.

 

An enhancement has been made to DB Distributor to allow better control on costing of Customer Owned Goods. When goods are initially received into the warehouse, the cost on the order line is frozen. If the AP Invoice has not been received, the cost used may not be accurate. The Sales Order Line cost at the time of the receiving is updated to the Unbilled Cost for the line. Later, when the AP Invoice is received, the cost may be different from the cost used to receive the goods. This modification allows the user to automatically update the order cost with the cost actually billed by the vendor. It also gives the user the option to update costs for historical presentation purposes as well.

 

During the PO Receiving Entry Program (P/O,P3), if the user is entering a receiving for a PO, without also entering the AP Invoice, and this first parameter is answered "Y", the program will determine if there is a potential costing issue. If the Sales Order Line meets the selection criteria, a "P", flag will be created in the PO Line flagging the potential costing issue. The user will get the following message:

 

"Cost Update Parameter Is Set - You MUST Receive AP Invoice BEFORE A/R Billing".

 

When the program advances to the billing area, the program will not allow the user to enter an AR Invoice for this or any lines on the order until the AP Invoice has been received and the PO Receiving Update has been run.

 

Later, during entry of the AP Invoice through PO Receiving Entry (P/O, P3), the program will check to see if the flag indicating a possible costing problem has been set. If the flag has been set, the program will check to see if the amount of the AP Invoice is different from the cost used to originally receive the goods. If there is a difference, the program will set a "U" flag in the PO Line, indicating that there is a cost issue that should be changed. In addition, the program will not allow the user to enter an AR Invoice for this or any lines on the order until the PO Receiving Update has been run.

 

The following message will be displayed:

 

"Unbilled Cost Update Parameter is set - MUST Update PO Recvg BEFORE A/R Billing".

 

During the PO Receiving Update (P/O, P4) of the AP Invoice, if there are costs that need to be updated, the following changes will occur if this first parameter is set to "Y":

 

The Unbilled Shipments Extension in the Sales Order Line Entry will be replaced with the Invoice Now Extension from Purchase Order Receiving Information. This is the cost amount overridden during entry of the Accounts Payable Invoice.

 

The program will determine the difference between the Unbilled Shipments Extension in the Sales Order Line Entry and the Invoice Now Extension from Purchase Order Receiving Information. It will then read through and accumulate all active Lot records in the Inventory Sub-ledger for receipts of this inventory item on this line of the sales order It will determine the proportion of the difference between the Unbilled costs and the amount billed on the Accounts Payable Invoice and redistribute the amount through the lots, based on the quantity in each lot. This will make sure that the full amount of the cost has been applied to the inventory lots.

 

If the second parameter is set to "Y", the following will also happen:

 

For the Unbilled Shipments, the UM Cost on the Sales Order Line Entry will be replaced with the Invoice Now Cost from Purchase Order Receiving Information.

 

The total cost amount previously written to the order booking file will be updated, in order to reflect an accurate booked profit amount.
