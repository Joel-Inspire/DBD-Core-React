Order Processing Parameters - Tab 2

 

DB Distributor > Order Processing > Setup > System > System Setup > Parameters (OPS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Feature driven pricing available?

If answered "yes", these fields will be available for selection in Custom Item setup. This is used for standard pricing by feature such a 2 color check, numbering, etc.
Should selling price be recomputed at invoicing (overruns)? If you are accepting overruns, do you need to recalculate your selling price at the time of invoicing?
Prompt for an item revision date in order entry? If set to "yes", the item revision date field will display with a prompt to enter the item revision date on each line of order entry.
Auto print new packing list for B/O items after billing? If you wish to maintain a manual file for filling backorders, answer yes. Packing list will reprint after sale journal update. Note: Back Order fill report and the Enhanced backorder fill report can do this automatically instead.
Set the default warehouse by division in requisition entry? Requires unique warehouse codes such as 0010 for div 10, 0020 for div 20, etc.
Print the complete special shipping address on the invoice? Will print the full address from special shipping in the lines section on the invoice versus the city and state.
Require line item discounts in order entry?

To enable the ability to enter in discounts by line item in order entry, set this to "yes".
Prompt for zero sell price during PO Receiving? If sell price is zero, the operator will be notified upon entering in PO in PO receiving. This will prevent zero lot sell prices from being written.
Require order acknowledgements to be printed? If the printing of order acknowledgements is needed, set this to "yes".
Eliminate backordered message line during invoicing? If an item is backordered, a message line telling the customer it is on backorder defaults to print on the packing list.
Interface the RFQ module with Order Processing? If you are entering quotes using the Request for Quote and would like to turn those quotes into orders, you will need to have this set to "yes".
Consolidate pick list code A= All Customers, S= specific customers , N = Not required. If not an “N”, then a new field will be available on the customer default data screen to allow for the gathering of pick list items to streamline pick and pack operations in the warehouse if desired.
Credit check warnings to be given during order entry P = "past due" check. System does other checks for over the credit limit, credit hold, etc.
Set default for SpeedBill to print packing lists/labels? If doing speedbill invoicing (creating order through invoice entry), do you wish to print a packing list and labels for the job? If not set to "no".
Shipping order acknowledgement If the printing of shipping acknowledgements is needed, set this to "yes".
Require user defined custom item specifications? Set to "yes" to be prompted for specs regarding envelopes, labels, etc. Must have the Electronic forms option to print the different templates.
Carton label format code varies by order line? If set to "yes", the carton label format field will be displayed in order entry lines.
Allow blank "date requested" field in order entry? If set to "no," the system will require the operator to enter in this date.
Force specification check if last order date prior to If a date is entered here, the system will force the user through the spec screens for any items who were last ordered before this date.
