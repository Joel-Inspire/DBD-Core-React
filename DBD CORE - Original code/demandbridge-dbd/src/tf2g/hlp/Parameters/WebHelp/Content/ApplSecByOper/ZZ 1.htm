Application Security by Operator - Tab 1

 

Setup > System Utility Functions > Administrator Function > System Access and Security> Appl Security by Operator (ZZX A6)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Restrict to the following T/M List Types

If you enter in telemarketing (contact management) list types here, the operator will only be able to view contacts within these lists. This feature restricts operators, typically sales personnel, from accessing information on accounts other than their own.

Allow delete option on Inventory Availability screen? This parameter should almost always stay set to "no". Setting this to "yes" will give the operator the option of deleting out an inventory warehouse location.
Restrict from changing salesperson and commission codes in order entry? Setting this to "yes" will let the operator see the salesperson code and commission code in order entry, but they will not be allowed to change them.
Prompt for "clear" option after Open RFQ report? If you check this parameter, the operator will be permitted to clear any RFQ data in the file after printing this information. We recommend that you set this to "no" for most operators and give this capability to your Sales Service Manager or other responsible key personnel.
Allow "balance" option for the GL Detail report? This option should only be used under the advice of DemandBridge support. If the G/L Detail Report Selection Screen Options field is set to BALANCE, then when the report is run, provided the operator has security clearance, the operator will be prompted to confirm they really want to run the report. If so, then the report will print showing the cleared detail as normal, BUT will make an entry back to the G/L Balance file equal to the net posting detail for the period. When the report is then re-run, the cleared detail will no longer show. This option should only be run to correct a data problem or following a data conversion.
Eliminate "continue" option after credit check message in O/E? If you check this parameter, you will not be able to continue with order entry if any credit warning is given. Since the time of this enhancement, DemandBridge has added two additional parameter options that are less restrictive. They are on tab 3 of this section.
Restrict an operator to the following AR division

If your company has multiple divisions, this provides you with the ability to restrict users to having access to their particular division only for most DemandBridge Functions. Enter the division code to which they should have access.
Restrict from changing the salesperson cost in I/C? Some companies use “Salesperson Cost” as a standard cost to be applied for the purpose of calculating commissions. If you check this parameter, it restricts the user from going into the item master file in Inventory Control and changing this cost.
Eliminate displaying of commission amount in Customer Setup? If you check this parameter, it restricts the user from being able to see salespersons commission amounts in the Open Invoice and Invoice History screens in the Customer Master file.
Restrict entry of orders with zero inventory available in stock? If you check this parameter, it restricts user from entering a release with zero inventory. In other words, it will not allow the operator to enter a backorder.
No entry of minus quantity for line with no PO code in Order Entry? If you check this parameter, it restricts users from entering a negative sales order and/or a negative requisition in Order Entry in order to bring inventory back into a warehouse.
Restrict from entering Stock PO's?

Setting this to "yes" will restrict the operator from entering a stock purchase order.
Restrict from adding new customers and deleting existing ones?

Setting this to "yes" will restrict the operator from setting up new customers and deleting existing customers.
Restrict from modifying customer master?

Setting this to "yes" will restrict the operator from modifying the data in the customer master record.
Restrict from adding new vendors and deleting existing ones?

Setting this to "yes" will restrict the operator from setting up new vendors and deleting existing vendors.
Restrict from modifying vendor master?

Setting this to "yes" will restrict the operator from modifying the data in the vendor master record.
