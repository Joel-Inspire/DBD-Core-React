Inventory Control Parameters - Tab 2

 

DB Distributor > Inventory Control > Setup > System > System Setup > Parameters (ICS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Do you group like inventory for quantity break pricing?

DB Distributor provides Pricing Tables that allow you to offer special pricing based upon the quantity of a specific item being purchased. If you have multiple items that should be considered as one for the purpose of quantity pricing, you should check this box. For example, you might sell 1099 forms for tax reporting purposes. Since there are multiple types of 1099 forms, during order entry, each type would be entered on separate order entry line. However, you may wish for the quantity of all lines to be taken into consideration to determine which quantity break is applicable.

Do you require more than 6 quantity price breaks? Checking this box increases to twelve, the number of quantity breaks in the Item file.
Automatic lot selection method code

Use the drop down list to select the method for automatic lot selection. The choices are:

Manual – No automatic lot selection. Operator designates which lot to use for the quantity being ordered.

Case pack then oldest – Automatic lot selection with emphasis on the case pack.

Oldest 1st, full cases – Automatic lot selection method where the oldest lot is selected, but the order quantity is adjusted to eliminate the splitting of a carton. This method will process through the list of lots in the order receipt date order and only full cases will be shipped. If a fractional case is required it will commit an additional FULL case. Therefore, it may ship more than was ordered. If more is ordered than on-hand, then ALL remaining lots will be committed and the program will revert to manual mode, giving the operator a chance to override this selection. Using the same data as the first example, with an order quantity of 5,600, three cases of 2,700 will be selected for a total order quantity of 8,100.

No Order Entry – This method does not commit lots during order entry. If you elect this method, you are assuming that the warehouse personnel will mark the accounting copy of the picking ticket with the information that accounting needs to relieve the appropriate lots during the billing function.

Lot display option

This parameter only applies for operators using the character version of DB Distributor Order Entry and/or inventory inquiry. It sets the preference as to which field is more important to have displayed on the main lot availability panel. The choices are:

The Distributor Purchase Order Number

The Bin Location

The Vendor Job Number

The Customer Purchase Order Number

The remaining fields would be displayed on the lot detail panel. In the graphical version of DB Distributor, with the use of a scroll bar, all fields can be displayed.

Ignore bin locations when printing count sheets? Generally Physical Inventory Count Sheets are printed with the items listed in bin location order. This facilitates efficient movement throughout the warehouse during the counting process. Check this box if you would prefer to print this report in strict item number order.
Number of months to use to calculate average monthly usage Enter the number of months for the average usage calculation. There are 3 figures that can be entered here. There are 3 different monthly average usage figures that can be calculated. As M1 M2 M3.
Should avg sales per month be displayed in I/C availability screen?

Adds an additional field on the inventory availability screen.
Prompt in P/O receiving if customer backorders? If there are customer backorders, the operator would be prompted that customer backorders exist during the PO receiving process.
P/O receiving update to change suggested bin location? If answered yes, the bin location entered in the lot record will be updated to the bin location field on the inventory availability screen.
Expanded lot record (std cost, u/m, storage exp date, PC and CC)

New fields have been added to the lot record. Turn this option on to make use of these fields. The new fields are as follows:

Standard Unit Cost

Standard Unit Cost Unit of Measure

Standard Unit Cost Quantity per Unit of Measure

Storage Expiration Date

Product Code

Commission Code

Lot Number

Display new lot numbers? Shows the lot sequence number on the inventory record.
Maintain a lot transaction log? A file containing Transaction Detail for Lots is now available. This file is the IC8 detail transaction file. This file is used in running the Detail Transaction report. The file contains information about each lot in the system and tracks receipts, sales/issues and adjustments. The file is keyed by Fiscal Year/Accounting Period, customer number, item number, and location. The information is updated during the P/O Receiving update, the Sales Journal update, and the Inventory Transaction update. Each entry in the file contains all of the information in the lot at the time of the transaction, plus some additional new fields. Also, the sales price is updated during the Sales Journal update.
Post inventory transfers to committed and on PO? Inventory transfers, while being entered in, will show the quantity on the inventory availability screen.
Display lot comment field during inquiry and lot selection? If this parameter is set to "yes", when lots display during order entry or inventory availability, a second line will display the lot comment field. If you view the lot detail, the line you chose will move to the top of the display (it has always done this). The comment field will display below this. This is used to display serial numbers for electronic equipment such as printers and PC’s as an example.
Automatic lot selection method for automatic reqs only

The program that creates Automatic Requisitions for eCommerce, Order Requisition Gateway, On-Line Services, Restocking Gateway, Restocking Report, etc. has been modified to check this new parameter. If blank the program will work as before, which uses the 'regular' lot selection method. If a valid method is entered it will be used for automatic requisitions only.

The main reason for this parameter is some users prefer to use manual lot selection over one of the automatic types. When auto reqs are created, if manual is the method, no lots get committed. With this new parameter, users can still use manual lot selection as their method for manual order entry, and have an automatic type method set up to use for auto reqs.
