Accounts Payable Parameters - Screen 2

 

DB Distributor > Accounts Payable > Setup > System > System Setup > Parameters (APS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Is there 1099 processing?

The system is capable of retaining information necessary for printing 1099 forms to some or all Vendors. Do you need to use this capability?
Current calendar year for 1099 processing If you are using the 1099 feature, enter in the current calendar for which 1099 information should be compiling.
Use extended terms code information? If you know that at least one of your vendors will allow a grace period beyond the invoice due dates, set this to "Y".
Allow vendor messages? Check this box to enable the Vendor Messages Feature. This feature allows you to enter up to four different vendor specific messages that will automatically display during Invoice Entry, Specific Invoice Selection, Order Entry, and Vendor Setup.
Allow special invoice amounts? When entering vendor invoices into the system, you may want to separate freight or other special amounts from the invoice total. If this is the case, check this box to "yes". You will then be able to access the Invoice Entry Options screen to setup which amounts you would like to track separately.
Is there automatic check reconciliation? If you will be using the Bank Reconciliation module, you will want to turn this on.
Activate intracompany GL account?

If you have your own internal production capability such as a graphic arts department or a print shop, you should check this box to enable intracompany vendors and P & L reporting for this profit center. This feature allows you to generate purchase (work) orders to and receive costs from your production departments. These receipts generate a revenue posting for the profit center while at the same time posting Cost of Sales for the Distribution side of your business.
Retain statistical data by salesperson? If you allow your sales staff to determine which of your vendors to use on any given job, you may wish to check this box so that you can get a report that shows vendor purchases by salesperson.
Use special invoice amount #2 for value added tax? This applies to distributors in Canada, Australia, the United Kingdom, or any other jurisdiction that uses a Value Added Tax System. It provides the ability to track Value Added Taxes paid, while the Sales Tax Reporting component of DB Distributor software tracks the taxes billed to the Customer.
VAT percentage If you set the previous parameter to "yes", then you will want to enter in the default VAT percentage here. This will automatically calculate the VAT amount when entering in invoices.
Reduce invoice amount by VAT for vendor purchase statistics? Check this box if you want your purchase statistics to be net of the Value Added Tax amounts.
In PO Receiving, substitute the division into the vendor category for GL interface? If you operate out of multiple divisions, you would probably want to have a different set of these accounts for each division. This parameter allows you to take advantage of the division substitution of accounts without the overhead of divisional account payable. If you operate multiple divisions, we suggest that you check this box and also the “General Ledger Accounts Interfaced by Category” parameter.
