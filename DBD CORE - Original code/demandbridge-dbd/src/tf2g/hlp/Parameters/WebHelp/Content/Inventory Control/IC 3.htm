Inventory Control Parameters - Tab 3

 

DB Distributor > Inventory Control > Setup > System > System Setup > Parameters (ICS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Default "lot inventory?" to "yes" for new items?

If answered "yes", the inventory master record will have the question of Lot Inventory automatically set to yes for new items being created.
Do you want file validation of QBC Groups?

If you answer this parameter "yes", you will need to set up the QBG Group Codes.

If you answer this parameter "no", the system will work as it presently does. You will be allowed to enter anything you'd like in the QBG Group Code field.

Presently, pricing in DB Distributor is setup to NOT use the QBG pricing if an Item Class/Price Category Table has been set up for the Item/Customer combination. If the validation of QBG Groups flag is set to Yes, the system will use the QBG pricing, overriding pricing from the Item Class/Price.
Create separate order lines if different lot sell prices?

This parameter applies to eCommerce and Order Entry Gateway Orders only.

The program that creates orders for both eCommerce and the Order Entry Gateway (OF2UAA) has been modified to send a flag into the automatic lot selection routine indicating that this new IC parameter should be checked and if set to Y then if more than one lot will be selected to fill the order line, then check the lot sell prices of the second and subsequent lots to be selected, and if the lot sell price is not zero, and the lot sell price is not the same as the sell price on the first lot, then exit back to OF2UAA temporarily, write out the order line, then come back in with a new order line to commit the next lot.

This new parameter will not change the logic concerning E-Logistics and backorders. This will continue to work the same as it does today.
