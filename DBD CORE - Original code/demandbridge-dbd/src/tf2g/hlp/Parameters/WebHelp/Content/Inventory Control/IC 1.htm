Inventory Control Parameters - Tab 1

 

DB Distributor > Inventory Control > Setup > System > System Setup > Parameters (ICS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
What size inventory code should be used?

DB Distributor offers the flexibility of identifying the size of your Inventory Code. Both numeric and alphanumeric codes can be used simultaneously, however, once this size has been set, it cannot be changed. Our recommendation would be setting it to 10.
Do you require customer specific inventory? Unless you sell only "stock" items, you will need to have this checked on.
Do you require a lot control sub-ledger to your inventory? If you require lot control in the maintaining of your inventory then set this parameter to 'Y'. In this situation, you may then identify at the Item level, which Items are to be tracked through lots. If you do not want to have lot control on any items in your inventory, then set this parameter to 'N'. This parameter must be set to yes if you wish to track bin locations, package qty, or different sell prices. This parameter must be set to yes if you have the eCommerce module. The default would be to have this turned on.
Enter code(s) of statistical data to be retained DB Distributor allows you to accumulate statistical data on your inventory items for reporting and inquiry purposes. You may select any or all of the available options listed above. In the next parameter, DB Distributor will require you to identify the number of prior years of each statistical data type to retain. Enter the codes corresponding to the statistics you want to accumulate side-by-side, or leave this parameter blank if you do not want to accumulate any inventory statistics. This parameter may be changed at a later date. The future addition of new statistic codes will begin the accumulation of the corresponding data. The future removal of existing statistic codes will cause the corresponding data to no longer be captured. The available options are: Dollar Sales, Unit Sales, Gross Profit, and Kit Unit Sales. We would recommend keeping all possible data. Hold the ctrl key on the keyboard and click to select multiple options.
Enter the corresponding number of prior years statistics to keep For each option selected in the previous parameter, enter in a number of years to keep the statistics.
Reserved for future use (DRD) Not in use at this time.
Do you require inventory transaction detail?

DB Distributor offers you the ability to record inventory transaction detail on your inventory items for later inquiry or reporting purposes. Transaction detail refers to all processing activities, which affect the quantity or cost of an inventory item.
Number of periods to retain inventory detail If you answered "yes" to the previous parameter, you must now specify the number of accounting periods that the transaction detail data is to be retained. This parameter may be changed at a later date. The future increase or decrease in the number of accounting periods to retain will take affect during the next period-end processing.
Do you require backorder fill reporting? Setting this parameter to "yes" will cause the Backorder Fill Report to be updated by the P/O Receiving update. This report can be used to identify those received items that are backorders with Customer Sales orders and bill as shipped orders. This report also allows you to designate that packing lists are to be reprinted for these backordered items.
Does this company post G/L inventory control accounts by location? If each warehouse location is to have its own General Ledger account number for inventory purposes, then answer "yes" to this parameter. This will cause DB Distributor to prompt you for an inventory sub-account number during the setup of warehouse location records. If you answer "no" to this parameter, then the next two parameter entries will be skipped.
Starting digit of the G/L sub-accounts number to substitute With a "yes" response to the previous parameter, you will need to identify the digit in your G/L account number where the sub-account begins. This may vary depending upon the format of your G/L account numbers.
Number of digits for sub-accounts In this parameter you will need to identify the number of digits in the G/L Sub-Account number, starting with the position identified in the previous parameter.
Reserved for future use (FDP) Not in use at this time.
Allow access to item code second description line A second description line can be entered for each inventory item. This additional description line will print on the customer invoice only.
Reserved for future use (BOC)

Not in use at this time.
