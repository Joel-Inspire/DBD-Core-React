Application Security by Operator - Tab 2

 

Setup > System Utility Functions > Administrator Function > System Access and Security> Appl Security by Operator (ZZX A6)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Password for operator to delete telemarketing contacts

If a password is entered here, this operator will need to enter the password to delete a record.
Restrict from deleting an order or line after packing list printed? Generally speaking, you would not want to delete an order or order line after a packing list has been printed, since the items might accidentally ship. You will want to have this set to "Y" on most users. An operator with the appropriate security setting can delete the order, after making sure it has not been shipped.
Restrict from changing or deleting telemarketing notes?

If you check this parameter, it restricts users from deleting or making any changes to telemarketing (Contact Management) notes. One thing that a company can do to preserve its contact with its customers is to maintain permanent notes on the contacts made over the years. This prevents the operator form changing or deleting important customer information.
Restrict entry of lines without a PO for C or I type lines in Order Entry?

Setting this to "yes" will restrict the user from entering releases from the warehouse.
Only allow name and address section changes in customer master? This can be set to "yes" if you have an operator that is restricted from modifying the customer master record, but you would still like for them to have the ability to change this basic information.
Allow entry to a closed fiscal year accounting period?

If you check this parameter, it allows user to make postings to a closed period. Normally the only person with this access would be your Accounting Manager or Controller.
Disable lookup ability at "send to user" field in send mail program? In the early days of DemandBridge, when email was not readily available, DemandBridge provided email capability. In the Electronic Mail system you had the ability to use the lookup feature and view all of the operator codes for the purpose of sending mail. If you had customers accessing your system via the Customer On-Line Services Module, you might not want them to see a list of your internal users.
Restrict modifying of lotted field and track lots for existing items? If you check this parameter, it restricts the user from going to Inventory Control and modifying the “lot inventory” field for that item. This field controls whether the system will track lots for the item.
Allow access to customer credit card option? If you check this parameter, it allows the user to go to the Customer Master file and set up or access client credit card information.
Allow access to intercompany GL accounts?

If you check this parameter, it allows user to be able to get to the “Intercompany GL Account” field on the vendor master record. This field is for the revenue account that would be used when the profit center for the company has a charge.
Restrict from viewing GP and Cost during order entry and invoicing?

If you check this parameter, it restricts the user from viewing the GP and Cost in Order Entry detail lines and in the Accounts Receivable Invoice Data Entry screen.
Reserved parameter

Not in use at this time.
Restrict modification and setup of terms code in OP and AR? If you check this parameter, it will restrict the user from modify the customer’s terms code in the header screen of Order Entry and in A/R Invoice Data Entry.
Disable "clear" option on Commission Booking and Payable reports? You will want to set this to "yes" for the majority of your users. Clearing the commission information removes the information from the file. This should only be done by your accounting manager or controller.
Restrict from deleting an order or line after PO printing? Generally speaking, you would not want to delete an order or order line after a purchase order has been printed, since it might have already been sent to the vendor. You will want to have this set to "Y" on most users. An operator with the appropriate security setting can delete the order, after making sure it has not been produced by the vendor.
Allow access to Scale Export Sequence No. in Req Entry? If you check this parameter, it allows access to the scale export sequence number field in the order header area for use with the DemandBridge Shipping Management module.
