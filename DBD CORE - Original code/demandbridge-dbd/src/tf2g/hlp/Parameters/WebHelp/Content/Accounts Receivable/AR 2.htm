Accounts Receivable Parameters - Screen 2

 

DB Distributor > Accounts Receivable > Setup > System > System Setup > Parameters (ARS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Do you age your AR by invoice due date?

Normally accounts receivable invoices are aged by the invoice date. If you want to age your receivables based upon the due date, check this box.

Age credit memos? If you would like for credit memos to be aged based upon the date of the credit memo, check this box. If not, all credit memos will be included in the current column.
Are your ageing categories in days? If unchecked, your accounts receivable ageing categories will be based upon accounting periods and the description on your ageing will be what you type in the Description field for each category. If checked, you will have the option to define up to 5 ageing categories based upon a number of days. For instance, if you wish to take action when an invoice is over 15 days old, you might want to check this box and make the following entries in the fields provided.
Ageing category to consider past due during credit check Enter the row number of the Ageing Category that you wish to trigger a past due warning.
Dollar range for payment analysis Enter here the three invoice dollar amount breaks for accumulating Average Days to Pay Statistics.
Report sales tax when an invoice is paid? The system can track and report sales tax either based upon invoicing or cash receipts. Check this box only if you report sales tax at the time of collection.
Value added tax?

Check this box if you are on a Value Added Tax System. This is for customers in Australia, the United Kingdom and Canada.
Next invoice number for automatic invoicing DB Distributor provides you with the option of automatically numbering invoices, either at the time of billing or at the time the customer invoice is printed. Enter your next invoice number here. This field will automatically update as you create invoices.
Renumber invoices during printing? When entering invoices, the delete option can cause invoice numbers to be skipped. By renumbering during invoice printing, you prevent the possibility of any breaks in your invoice numbers. Note: By checking this "on", your invoices will always be renumbered. The number set in the previous parameter basically becomes a temporary invoice number, and the number set in the next parameter will become the final invoice number.
Next Renumber If you are using the renumbering option from the previous parameter, you will need to set the number range here that will be for the final invoice numbers.
