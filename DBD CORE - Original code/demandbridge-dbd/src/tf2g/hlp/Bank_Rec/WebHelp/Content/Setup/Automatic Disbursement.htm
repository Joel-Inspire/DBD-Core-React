Automatic Disbursement

 

DB Distributor > Bank Reconciliation >Setup > Automatic Disbursement (B/R S3)

 

The Automatic Disbursement Utility is used to import the existing disbursement data from Accounts Payable for all active bank accounts. You cannot start processing in the Bank Rec module until you have run this utility. Before running this utility you will want to set your system accounting date to the first date of the period that you will first be reconciling. For example, if January 2009 will be the first period that you will be reconciling, then you would want to set the accounting date to 01/01/2009. When running this utility, all checks dated within 90 days prior to the accounting date will be marked as "cleared". If you have checks within this range that have not actually cleared the bank, you will be able to "unclear" them. Also, if you have items still outstanding from before the 90 day period, you will have the option of adding them as outstanding through the Detail Inquiry Utility. Deposits in Cash Receipts will only be recognized after the running of this utility.

 

This is the utility referred to in the 3rd parameter from the parameters screen. If the parameter is set to "no", then when you enter this screen, it will ask you if you would like to run the utility. If the parameter is set to "yes", then when you enter this screen, it will notify you that the utility has already been run, and that you would need to reset the parameter in order to run it again. Note: This utility should not be run after the original loading of disbursement data and entry of bank reconciliation information.
