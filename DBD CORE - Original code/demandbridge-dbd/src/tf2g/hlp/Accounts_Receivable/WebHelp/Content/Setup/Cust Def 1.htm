Customer Default Data 1

 

The customer default data is used to help populate information about the customer when working in other parts of the system. This information is spread across multiple panels. Below is a listing of the available fields.

Field Description
Is customer PO number required? Turning this on will require that a Customer PO be entered in the Header area of Order Entry for any orders for this customer.
Unique req? If the previous answer is "yes", do you require that the Customer PO be unique?
Thank you letter type default As a by-product of order entry, you have the option to send sales confirmation letters. The letters can vary from customer to customer, or even order to order. Enter the single character code for the letter that you would want to send to confirm orders placed by this customer. Note: “N” means no confirmation letter. Standard letter is “A.”
Which Orders "A" for all orders get a thank you letter, "M" for only non-eCommerce orders get a thank you letter.
Normally print shipping acknowledgements If checked, then shipping acknowledgements will automatically be queued as a byproduct of job status posting.
Freight Markup If you want to set a standard freight markup with this customer, enter it here.
FOB Designate if you want to use a standard Freight on Board code in the Shipping area of Order Entry.
How to Ship Designate if you want to use a standard How to Ship code in the Shipping area of Order Entry.
Inside Delivery Enter in if you want to have a standard Inside Delivery set in the Shipping area of Order Entry.
Ship Via Designate if you want to use a standard Ship Via code in the Shipping area of Order Entry.
Finder's Fee Type If you would like to pay a finder's fee with orders for this customer, select one of these options: Percentage of Sales, Percentage of Gross Profit, Percentage of Cost. Adjust Gross Profit, or None=no finder’s fee at all
Vendor The finder's fee will be paid to a "vendor" code. Select the vendor that the finder's fee should be paid to.
Rate Enter in the rate to be paid, based upon the finder's fee type previously selected.
Summary Bill for If this customer is to be a summary bill customer, you can select up to 3 invoice types that should be a part of the summary bill. The options are "I" for invoices of Sales Orders, "R" for invoices of Releases, and "W" for invoices of Billable Releases.
Print Invoice for Each Shipment For the options entered in above, enter a "Y" or "N" under each one to designate if you want the individual invoices to print in addition to the summary bill.
Monthly Shipment Detail Reporting

We recommend that you check this box. It provides shipment information by customer location, customer cost center and by warehouse. This affects the following reports:Warehouse Release Report, Monthly Activity By Warehouse, Monthly Usage Report, and Shipment Detail by Location- for summary bill report.
Consolidate Pick List If you check this box, requisitions and warehouse releases will update to a file so that a Consolidated Pick List can be printed. This report will list items to be picked in bin location order so that a single pass through the warehouse can be made.
Set No-print flag to yes for 9### releases on W type orders On eCommerce orders, customers may order products that are customer owned along with items that are distributor owned. In other words, they are mixed on the same order. You may wish to invoice the distributor owned items and use the summary billing program to report to your customers the items that are customer owned.
Update FMZ file

Assuming the above parameter is checked, these same transactions should be updated to the FMZ file. We recommend that you check yes for this option.
Do you wish to treat this customer as an FMS customer? This is used to tell the system if you want to designate this customer as being a Print Management customer, formerly known as Forms Management (FMS).
Format code for reorder notices If you want to use reorder notices with this customer, enter in a valid reorder notice format code to use.
Customer always gets lowest QBG price? If using quantity break pricing, should this customer receive the best price, regardless of the quantity ordered?
Reduce on-hand by committed qty? On reorder notices, an on-hand quantity is displayed. Checking this box will reduce the on-hand amount by any open orders.
Enter Default FMS Method for Locations If the customer has been designated as a Print Management customer, you will need to specify if they are "issue based" or "counted".
