Form/Item Image Requirements

TopForm WebEC currently support five (5) images per form that you place on the web. These images may be used either for simple image placement or as an image source in a hyperlink. Form images are currently available for display on the following pages.

  • Product Catalog

  • Product Catalog Detail

  • Item Search Results Page

The following sections list the data tags that may be used to identify the form image to be used on these pages.

Simple image display

The following data tags can be used for simple image display:

  • Image_form1

  • Image_form2

  • Image_form3

  • Image_form4

  • Image_form5

  • Image_form1_link

Hyperlink image source

The following data tags can be used for display of an image as an image source in a hyperlink:

  • Image_form2_link

  • Image_form3_link

  • Image_form4_link

  • Image_form5_link

Form Image Name Specification

The form image file names can be set up on the WebEC form parameters screen. On this screen you will see five (5) fields for specification of the form image name. Each form image field (1, 2, 3, etc.) corresponds to the data tags described above. These file names will be appended on to the image path (either from the customer parameter record if separate image directories are desired or from the WebEC system parameters).

User Definable Form Image Labels

As a part of your TopForm WebEC system, you can define the purpose of form images being used on your web commerce pages. These purposes might include a thumbnail, a low resolution graphic, a high resolution graphic, an icon, etc.. By defining labels for the five (5) form image fields, you won’t have to remember which image is for what purpose.

To define the labels, modify your WebEC system parameters and enter in the labels.

Maintenance of Form Images

When we are speaking of multiple images supported for a single form for different purposes, each of the image files will contain the following format:

<image file name> + <image suffix>

The image file name is either going to be the customer code plus form code or simply the form code. The image suffix will contain an identifier for the file purpose (i.e. lr for low resolution) as well as the full extension based on the type of graphical file (.gif, .jpg).
