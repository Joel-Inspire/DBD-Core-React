Customer Order Maximums

Web.ec supports order maximum limits by ship-to location using a variety of control levels. These levels include the following:

  • Percentage of average monthly usage (see below)

  • Fixed monthly quantity set globally at the item level

  • Fixed monthly quantity set individually by product or by ship-to

Activating the Feature

The order maximum activation control is done at the customer level in the Customer Parameter Setup. This enables you to turn on this feature only for those customers who require limits and relieving webec from processing this additional information for accounts that do not need limits.

Percent of Average Monthly Usage

Percent of Average Monthly Usage is specified in the Buyer Setup and allows you to specify a percentage of the average monthly usage to use as the order maximum limitThis feature allows you to evaluate ordering for between 1 and 12 prior months.

Example:

Let say you are setting a monthly maximum for all locations at 2,000 per and you have an entry of 3 in this field. Web.ec will then look to see the total usage for the ordering location over the last 3 months (lets say it was 3,500). Webec will then allow 2,500 more to be ordered.

  • Monthly Max:  2,000

  • Accumulate:  3 months

  • Total Available Usage:  6,000

  • Usage-to-date: 3,500

  • Remaining:  2,500

Note:  If this field is left blank or set to 0 or 1 then order maximums will be monthly. There is a maximum of accumulation over 12 months.
