Web.ec system components

Web.ec is the software tool used to publish your TopForm product catalog on your web site, and the system that enables your buyers to complete online purchasing transactions. The following tables list and describe the major Web.ec system components. See Web.ec transaction processing for information on how these Web.ec system components handle Buyer transactions.

Web.ec Component

Purpose / Use

TopForm Server

System platform that supports the TopForm application suite.

TopForm Web.ec module

Software module used to set Web.ec system operating parameters and publish Buyer catalogs on the web site.

Web Server

Apache or Windows IIS web server that serves the Web.ec pages to the Internet

TopForm Web Server (ecserver.pl)

Server process (Perl-CGI script) that receives and processes client requests (in the form of a #.ec file), and serves the Web.ec web pages to the client. Resides on the TopForm tf2w directory in the following path:

[TopForm Directory]/tf2w/EC3/perl

TopForm Web Client

(ecclient.pl)

Client process (Perl-CGI script) that takes requests from the Buyer and passes them on to the TopForm Web Server process. Resides in the /cgi-bin directory on the Web.ec web server.

/usr/lib/pvx

Typical location of the TopForm server and application suite on ProvideX systems. Other Unix systems will vary.

tf2w directory

Subdirectory in the TopForm server directory (/usr/lib/pvx or equivalent) that contains the Web.ec program files.

www_interface directory

Subdirectory in the TopForm server directory (/usr/lib/pvx or equivalent) that contains the Web.ec template pages and #.ec data files.

#.ec lead-in parameter files

Files created by the ecclient.pl process to call the ecserver.pl process and pass the appropriate "lead-in" parameters based on the page requested by the Buyer. See Web.ec transaction processing and Web.ec page templates and tags.

#.ec.o output data files

Files created by the ecserver.pl process to serve output to the ecclient.pl process. See Web.ec transaction processing.

Web.ec basic programs

Programs that use the embedded Web.ec page tags to dynamically access and retrieve TopForm data, and generate the output used for the web page. For example, EC3PC0 is the basic program that generates and controls the Product Catalog page, while EC3WS1 is the basic program that generates and controls the Order Entry page. See Web.ec basic programs for more information.

Web.ec page tags

Special tags whose parameter values control the loading, operation and data-retrieval on Web.ec pages. See Web.ec page templates and tags for more information.

Web.ec page templates

Web pages embedded with Web.ec page tags that are provided by TopForm when you purchase Web.ec. See Web.ec page templates and tags for more information.

logon.htm

Default web page template used for Buyer login. Resides in the root (/) directory on the Web.ec web server.

Images directory

Directory that contains the images used on Web.ec web pages for catalogs, page templates, and company logos. See Web.ec image management for more information.
