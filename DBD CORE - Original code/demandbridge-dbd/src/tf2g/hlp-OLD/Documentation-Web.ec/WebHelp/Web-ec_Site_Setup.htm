Setting up your Web.ec web site

Web.ec serves as a link between your web site and your TopForm system. Setting up this link and adding Web.ec functionality to your web site requires the completion of the following actions. See Web.ec system components and Web.ec transaction processing for more information on these tasks.

  1. Set up the Web.ec client program (ecclient.pl) received from TopForm on the Web.ec server (Apache or Windows IIS server that will serve the Web.ec pages).

  2. Load the Web.ec login page (logon.htm) received from TopForm onto the root directory of Web.ec server .

  3. Have a TopForm support representative configure your TopForm server to support Web.ec. A TopForm support representative will create the following on your TopForm system directory (/usr/lib/pvx or equivalent):

    • Web.ec programs directory (tf2w)

    • Web.ec interface directory (www_interface or similar) that contains the templates and data files used by the Web.ec programs

    • Web.ec development directory (www_dev or similar) used for customizing your site design without impacting the "live" site. This directory contains copies of the www_interface templates and data files.

  1. Create and load images into the designated image directory. See Web.ec image management for more information.

  2. Customize your Web.ec page templates according to your requirements. See Web.ec page templates and tags for more information.

STOP. You have completed this operation.
