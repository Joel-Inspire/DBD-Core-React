Web.ec transaction processing

The following steps describe how the Web.ec system processes transactions. See Web.ec system components for more information on the components mentioned below.

  1. A Buyer arrives at the Web.ec logon.htm page on the distributor web site and attempts to log in.

  2. The Web.ec client program (ecclient.pl) generates a randomly numbered data file (#.ec) containing the Buyer login information and template instructions.

  3. The ecclient.pl program initiates a CGI call and passes the #.ec file to the Web.ec server program (ecserver.pl) on the TopForm server.

  4. The ecserver.pl program writes the #.ec file to the Web.ec interface directory (/www_interface), and launches the Web.ec server process (EC3SRV).

  5. The EC3SRV reads the "lead-in" parameter values in the #.ec data file and completes the following actions:

Lead-In Parameter

Purpose / Use

user=[buyerid] and password=[secret]

Authenticates the Buyer login.

template=[template.htm]

Determines and retrieves the required template page for this buyer. For example, if prdcat.htm is the name of the product catalog template for this buyer, then template=prdcat.htm will retrieve the product catalog template page and associated images.

basic=[data retrieval program]

Identifies and launches the appropriate data retrieval program that uses the Web.ec page tags embedded in the template page to dynamically access and retrieve TopForm data, and control web page interactions. For example, basic=EC3PC0 will launch the EC3PC0 program that controls the creation of the product catalog page.

  1. When its processes are complete, the EC3SRV program places the output file (#.ec.o) in the  /www_interface directory, and erases the #.ec file.

  2. The ecserver.pl program "sees" the #.ec.o file and retrieves the output to generate a web page, which it serves to the ecclient.pl program. The EC3SRV program then erases the #.ec.o file.

  3. If login was successful, the Buyer will see the page designated by the values set by the "basic" and "template" lead-in parameters. If login was not successful, the Buyer will see a "relogin" page that you set up and designate in the Web.ec System Parameters setup.

  4. After logging in, the Buyer navigates around the Web.ec pages by selecting the available buttons, links and forms. Each navigational device must contain the appropriate lead-in parameters to allow the Buyer to properly view the next page. See Web.ec page templates and tags for more information.
