Server Parameter Maintenance dialog fields and settings

Clicking the "Server Setup" item from the Server menu launches the Server Parameter Maintenance dialog. The Server Parameter Maintenance dialog is used to view and modify server settings. The following table contains the field descriptions and the setting options and requirements of the Server Parameter Maintenance dialog.

Field Name

Purpose / Use

Setting Options and Requirements

Server ID

Enter the ID for how the server will be referenced by the system.

Any 15 character name, but Cannot contain blanks or special characters (*, ?, !, etc.)

Server Description

Enter a text description to help identify the server function.

 User-Determined

Server Directory

Directory on your TopForm server where requests will be processed. Generally this is a directory in your “/usr/lib/pvx” directory that is prefixed with “www_”

Examples:

/usr/lib/pvx/www_interface

/usr/lib/pvx/www_dev

File Mask

The file suffix is determined by the Perl script being used to service this server.

DA:*.<file suffix>

Log File

The server will update this log file with all activity messages

Any file name, but cannot contain blanks or special characters (*, ?, !, etc.)

 

Base Page Ref

This is the base IP or URL for the web server servicing this configuration. This reference will be used to replace the data tag ?tf?base_page_ref? on your web page templates.

Correct web server IP/URL
