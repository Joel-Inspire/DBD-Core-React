Web.ec Sort Code Types

Currently you have the ability to define Sort Code parameters according to the following types:

  • Order Status Sorts

  • Ship-to Drop Down List

  • Customer Specific Ship-Via Drop Down List

  • Email Subject Lines

The following table defines each of these Sort Code parameter types.

Sort Code Parameter

Purpose / Use

Order Status Sorts

Order status sort codes control the sequence in which orders are displayed on the order status page.

Ship-to Drop Down List

 

Ship-to drop down list sort codes define the format of the ship-via list that is displayed for a customer. The list is automatically sorted and displayed based on the data elements that you define in the UD Sort Element Definition dialog. It is recommended that if there are identifying codes on the ship-to (cost centers, department codes), that these be used as the initial elements of the definition.

Customer Specific Ship-Via Drop Down List

Customer specific ship-via list sort codes control the ship-via’s that you want to show each customer's buyers that log onto Web.ec. Once you have set up a single ship-via code in the UD Sort Element Definition dialog, the feature is automatically activated. If no codes have been set up for the customer then the system will show all available ship-via codes that have not been restricted from the web in the TopForm Order Processing module (O/P System Setup, "Ship Via Code Setup").

Email Subject Lines

Email Subject Lines sort codes control the subject line for automated e-mail action codes. You define the subject text by customer or for all customers using the UD Sort Element Definition dialog.
