Web.ec E-Mail Server Configuration

The following sections describe the steps to configure the automated e-mail functionality of Web.ec.

  How it Works

This discussion uses the automated emails for order purchases as an example which results in an email of the order confirmation page to the flagged email addresses.

 

Action

Process

Result example

Order Placed

WebEC will create a copy of the order confirmation page and will place this copy in the active “www_interface” directory. As part of this process, the email packet header and termination will be placed at the beginning and end of the resulting page. One copy will be made for each email that is flagged with an “O” for the buyer who placed an order. These files will be prefixed with “TMP”+<random number>

TMP4562

TMP51999

Email Files Generated

Following the copies of the email files (TMPxxxx), WebEC will generate email request files in the active “www_interface” directory. These files have an “*.em” suffix and are the files that are processed by the WebEC server. One file is generated directing the email of each of the TMPxxxx files discussed earlier.

 

File Example:

basic=EC3EM0     

template=TMP35984

session_id=0353922

remove_file=Y    

11280.em

32453.em

Emails Sent

The TFWS Email server will find the “*.em” files and will email each file, first come, first serve, using the following command:

 

/usr/lib/sendmail –t < path_to_www_interface/TMPxxxx

Email Server Setup

You will need to be running the TFWS 3.0 (TopForm Web Server) that supports the set up of multiple servers. Automated email functions will generate files that are placed in the active “www_interface” directory. All files are placed with an “*.em” extension, regardless of the configuration (ie. no *.em_dev for development servers, etc.).

 Email Subject Line Definition (U1/U2)

The subject line for the automated emails is configured using the user definable code tools of U1 and U2.

 

1.      Email Subject Code Setup

The code to be used for the email must begin with “EMAILSUB_”. The final character will correspond to the action code being supported (currently, this must be O). Following the addition of the code, the remaining field entries will default to the appropriate settings. Press F4 and exit.

Email Address / Action Codes

The email addresses processed are set up on the individual buyer’s records. The action codes supported include the following:

 

Action Code

Description

O (letter O)

This will email the order confirmation page to the email address.

 

‘sendmail’ Configuration on TopForm Unix Server

Your Unix system comes pre-loaded with a sendmail option. This configuration of this option needs to be configured to route outgoing email messages routed to your email server. This configuration will be done by TopForm Technical Support

 

Note:  All configured systems need to have the “sendmail” command loaded into the “/usr/lib” directory
