Catalog Category Naming

The Catalog Category field determines the order in which the Catalog Category Description will appear on the Product Catalog drop down list. This field can be a maximum of 10 alpha and numeric characters but it will not appear on the web pages.

Customer Naming Requirements

A customer may want to identify and group their forms into categories of their own choosing. For example, instead of "flat sheets" a customer may want call these products "General Ledger Forms." You define this by the name you assign to the category in the Catalog Category field of either the Product Catalog Setup or the Product Catalog Load Utility.

 

Category Sequencing

The Catalog Category field determines the order in which the Catalog Category Description will appear on both the Product Catalog and the Order Entry web pages. The customer may want to decide the order in which the list will appear. This can be accomplished by making the Catalog Category numbers a combination of up to 10 alpha and numeric characters (such as 100GENLEDG, 200CASHTCK, 300OFFSUPP, etc.). Using large numbers will also make it possible so that if additional categories (such as 101, 102, 103, etc.) are needed at a later time, they may be easily added in between each major category.
