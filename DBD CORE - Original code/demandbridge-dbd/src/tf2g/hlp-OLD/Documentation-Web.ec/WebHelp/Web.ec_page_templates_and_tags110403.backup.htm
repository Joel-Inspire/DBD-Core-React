Web.ec page templates and tags

Web.ec operates using special tags that control the loading, operation and data-retrieval of Web.ec pages. When you purchase Web.ec, TopForm provides template pages embedded with these tags. You can completely customize the graphical "look and feel" and rename of the template pages according to your requirements. However, the Web.ec page tags must be used on these pages according to TopForm specifications.

Template Page Tag Types

The following table lists and describes the types of page tags used on the Web.ec template pages. See the document called TopForm Web.ec Web Template Documentation (available on the TopForm web site) for the complete specifications of the Web.ec template page tags.

Template Page Tag Type

Purpose / Use

Lead-In Parameters

These parameters are passed via a CGI call from the ecclient.pl process to the ecserver.pl process in the #.ec file.  Lead-In Parameters define the parameters required to link to the requested page such as the Web.ec basic program and template used to generate the page. For example, Lead-In Parameters basic=EC3P0 and template=prdcat.htm would generate the Buyer's product catalog using the prdcat.htm page as the template. See Web.ec system components and .

Page Control Tags

These tags are used to identify sections of the page for replication, heading identification, etc. Examples of Page Control tags include the tags for table layout,  drop-down lists, buttons, and links.

Data Tags

These tags are used to define the parameters for the data retrieved from the TopForm Server.
