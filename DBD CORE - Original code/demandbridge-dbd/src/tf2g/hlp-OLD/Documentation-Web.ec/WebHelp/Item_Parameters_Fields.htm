EC Item Parameters dialog fields and settings

The EC Item Parameters dialog is used to control the parameter options for individual items within Web.ec. The following table contains the field descriptions and the setting options and requirements of the EC Item Parameters dialog.

Field Name

Purpose / Use

Setting Options and Requirements

Customer Code

Enter or select the Customer Code. Customer codes are set in the TopForm Accounts Receivable module.

Valid TopForm Customer Code

Item Code

Enter or Select the Item Code. Either the non-existence of this record or not selecting any categories will cause this Custom Item to be excluded from the catalog when it is refreshed using the catalog load utility.

Web.ec Item Code

Catalog Categories (20 Max)

These fields allow you to identify which categories this Item will appear in. Forms/items may appear in up to twenty (20) categories simultaneously. If no categories are assigned, then the Item will not be loaded when the catalog load utility is run.

Note: If the category you need does not appear in the list, you must first go through the steps necessary to set it up using the Catalog Category Setup and then return to this step.

Web.ec Category

Location Monthly req max

If the customer has been activated for order maximum limits then the quantity listed in this field will be used as the global maximum for all ordering locations (ship-to’s) that do not have order maximums specified at the location level using the Custom Item Setup item of the Setup menu in the Order Processing Module. If this entry is set to zero then this order maximum level will be ignored.

Integer

Warehouse Group

This option is available with the licensing of the E-Logistics , and allows you to specify a warehouse group set that should be used for inventory selection for this item. This entry will override the warehouse group that has been set up on the associated customer.

See E-Logistics documentation

Pricing Source

This entry instructs Web.ec from where to obtain pricing for this product.

Setting Comments

C = The pricing will be pulled from the custom item master. This option disables the use of quantity break pricing or special pricing tables that can be set up only on the inventory item master for an itemNote: This value cannot be set for stock inventory items.

I = The pricing will be pulled from the inventory item master facilitating the use of quantity break pricing and special pricing tables. This setting requires that your Web.ec product catalog page template use the <!?tf?qtybreakstart?><!?tf?qtybreakend?> data tags for display of pricing information.

Note: If at the time the product catalog is displayed and an item is found to have the “I” setting for the pricing source yet no inventory master record can be located, then the product catalog program will automatically use the custom item master for the source of the item.

C / I

Create P/O on Order?

This field is used to identify the item being ordered is to generate an exact repeat for the item. The system will then look to the Web.ec System Parameters to get a range of P/O codes to use on the order.

Checkbox

Secure Codes

Each item can be assigned up to five (5) two-character secure codes. If an item contains secure code entries, a Buyer will only be able to see the item in their catalog if either of the following is true.

  • The Buyer has super user secure code status. This is set up on the Buyer Setup

  • The Buyer has the one of the assigned two character secure codes on their Buyer Setup.

Item Selection Codes (1-15)

These entries are designed for Stock inventory items only. Each item may be assigned up to fifteen (15) selection codes. These codes are used to describe or classify the item and is used in the selection of stock inventory items in the loading of stock catalogs using the catalog load utility.
